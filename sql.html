SQL:

What It Is

We just recently covered SQL, its not something that a class would cover for the entire year but it is still plently useful for data science researchers and developers. SQL stands for Structured Query Language and it is used to communicate with a database. SQL queries(statements) can be used to update a database or mostly in our case, retrieve data from a database using the SELECT statement. The syntax for the language seems to be relatively simple, but the hard part about SQL is thinking of what you need to do to pull data off the database. Also remember that we covered beginner topics mostly.

Coding in SQL:
Install

The first step in being able to code in SQL is obviously checking if you have it installed. If you already have python3 installed, which you most likely do, then you should also have sqlite3 installed. Check if you have it installed by running sqlite3 in your [Terminal or WSL]. If you don't have it installed, follow the steps here to do so.

Running SQL

You can run SQL on either your Terminal or WSL, so there's no need to worry about installing those. But if you haven't installed, then look at the Getting Started page to do so. Once in your designated shell, you can run sqlite in two different ways. One, you can run sqlite3 and then just input your code directly into the line, or two you can use a text editor and create a program file and then run that as a query. More information on that here.
