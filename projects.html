Cryptography

This cryptography project was our first project of the year, where we had to cipher some different input text with three different cryptosystems, Caeser Cipher, Vigenere Cipher, and the Merkle-Hellman Knapsack Cryptosystem. The Caesar cipher was easiest to implement while the MHKC cipher was the hardest. This project was just meant to act as a jumpstart to the class, where we could touch up on some Python skills or learn new ones:

NPuzzle

NPuzzle was our second project of the year and the difficulty also proved to increase as we did this project. Basically this project was a puzzle, a 15-square puzzle that had numbers 1-15 in a randomized order. Learn more about that here. The goal of our project was to create a solver for this puzzle, meaning the program takes the puzzle and then organizes the numbers top-bottom left-right until there is a hole in that last square. Through that, we had to use three different search algorithms that would solve the puzzle and then also optimize them to make them fast:

    2. Our solution: Solutions. [Note, do not run DFS it will crash]

Percolation Game

The Percolation game was a sort of contest between the ATCS class where we had our teams build this game where players can play against one another in a vertex-removal game. What that means is we have vertices from a Binomial Random Graph, and the goal of the game is to remove all the vertices. A player loses the game once there are no more vertices of their color to remove. The way the program worked is that we had a benchmark of a "player" that randomly chose to remove one of their vertices at a time, and our goal was to make the program beat said "player". The contest was, whoever had the highest probability or efficiency of beating the "player" was the winner and won a gift card:
    1. Official project description: Percolation Games. Try it out.
    2. Our solution: Solutions
    3. Percolation Theory

Build-Your-Own-Shell

The Build-Your-Own-Shell project was by far the hardest project of the year. The shell you use is essentially what we had to build on a much smaller scale. We had to make a Linux shell using Python that could run commands like cd, exit, pwd, jobs, fg, and bg. The behavior of our Linux shell had to match the /bin/sh program we can run on our shells. This project was the hardest because there were a lot of new things that we didn't know about. As a result, this project required a lot of time to research how Linux shells work and how the code works. Not only that, but all Linux shells could be different, and there is a lot to implement, so for this project we had to create a Design Document. Basically what a Design Document shows is all the details we want to implement into our program, such as the design of our shell and the optimization of it. Even though it took some work, the design document helped a lot in writing the code:

Final Project

The Final Project was, as the name suggests, our final project for our year in ATCS. And you're looking right at it. The purpose of our final project was to learn HTML and CSS because they are useful to know and then do some exercises with them. But instead of doing an exercise, we decided to make this website to recap our year in ATCS and display some resources that we used to succeed in the class. The reason that we chose to do an ATCS Guide is because we wanted future students to have a support system that they can use to kickstart their journey through ATCS. With this website, students can look at our past projects, as well as learn the basics of HTML/CSS, Python, and SQL.
