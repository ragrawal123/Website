Installing Python:

Mac

Go to Python.org and select Download Python 3.9.5 for Mac OS X.

Windows

Go to Python.org and select Download Python 3.9.5 for Windows.

What to Run it On:

Mac

For Mac, there is no need to install anything else to run your programs on. Terminal is already installed for you and you can use that to run your programs. The steps for running a program in Terminal are
1. Open Terminal.
2. To check that Python is installed, run the command python3 and the terminal should go into a Python shell with the characters >>> showing. To quit out of the Python interactive, type in quit().
3. cd into your Github repository containing your program file[Github stuff down below].
4. ls to make sure your program is in the repository (doesn't need to be done all the time).
5. Run the program using python3 programName.py.

Windows

Windows is a bit more complicated. It doesn't inherently have a nice system to run on like Terminal, but we can install one. Here we are going to install the Windows Subsystem for Linux. First what we want to do is install the Linux subsystem. The steps to do so can be found on the Microsoft Docs: Install WSL. We recommend using Ubuntu 20.04 when it prompts you to choose an app to run the subsystem on from the Microsoft Store.

Once you are done installing, you will need to open the Ubuntu app up and create a username or password. Keep in mind that when typing in the password it doesn't show anything, but the characters are there so be mindful. Once done, check that Python is installed by running the command python3. The terminal should go into a Python shell with the characters >>> showing. To quit out of the Python interactive, type in quit(). You should be able to then cd into your desired directory and run the program using python3 programName.py. Again, to check whether the program is in the directory you can do ls.

Github:

Github is a space for programmers to collaborate with eachother and work together on the difficult projects that you will have to take on in the ATCS class. Github is a space where you can upload the files of your code so that you and your team can look at it without having to ask the initial coder to email you the files. Basically, you or someone else on your team can code on their own time, and then upload that code to the repository for others in the team to look at later. There are multiple steps to get set up on Github, here is what they are:

1. Go to Github and sign up for an account. If you already have one log in.

2. Once done, at the top left corner you'll see the word "Repositories". On the right of that click the New button to create a new repository that you will use for your code. Create a repository name that represents what type of project you are coding. On the bottom check the box that says "Add a README file" because you can edit that and use it as a design document or an explanation of your project. Create your repository.

3. This part gets a little complicated. Open whatever you use to run your program[Terminal or WSL]. Here you now need to clone your repository to your desktop so that you can cd and make your program file there. To do so you will need to go into your repository on Github, and under the tab "Code" you will need to copy the link that is there. Go back to your [Terminal or WSL] and run the command git clone [copied repository link]. After it's done cloning, go into the repository by running cd [repository name].

4. After cd-ing into your repository, you want to create your program file. To do so, run touch programName.py. Note that it does not have to be a python file, it could be java or html or css etc. To check you created the file in your repository, run ls. You now created your program file and you can use your choice of text editor to get started! Remember that when you are saving your program file for the first time, save it in your repository.

5. After doing some coding, you probably want to upload your file to github so that your last commit is saved or your teammates could also look at the code. First make sure you are in your repository in your [Terminal or WSL] and ls to make sure your file is there. You then want to run these 3 commands in order:
    1. git add fileName
    2. git commit -m"some message"
    3. git push
It will prompt for your username and password, but after that your file should be on Github!
